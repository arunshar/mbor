Multi-level Bi-objective Routing

An independent Rust reimplementation (and GPU acceleration) of Multi-level Bi-objective Routing (MBOR): find the complete Pareto-optimal set of paths under two costs (time and energy), fast, by precomputing a boundary multigraph and pruning with two-dimensional cost-intervals. It reproduces the paper's speedup on open data, and every result is exactness-checked.

Abstract

The bi-objective routing problem asks for the full Pareto frontier of paths between two points in a graph whose edges each carry two non-negative costs (for example travel time and energy). The set can be exponential, so compute-on-demand search is slow. MBOR precomputes a boundary multigraph and a Multi-level Encoded Pareto Frontier View (MEPFV), then answers a query by combining a few precomputed Pareto sets, with two-dimensional cost-interval pruning to skip combinations that cannot contribute. This repo is a from-scratch Rust port that reproduces the method on the open 9th DIMACS Bay Area network, plus Triton GPU kernels for the dense sub-steps.

Why it is trustworthy. Every MBOR answer is checked against two independent exact baselines (BOA* and bi-objective Dijkstra) and an exact full-graph search. On real Bay Area queries the four methods return the identical Pareto frontier (zero mismatches), and the average solution count matches the paper.
Honest scope. Two things stated plainly. (1) Rust is not faster than good C++ at the language level; the value here is a memory-safe, one-command-reproducible build that reproduces the paper's speedup on open data. (2) The online retrieval is reproduced; the precompute is correct but not yet memory-optimized for the full BAY network (it stores the full fragment Pareto view). The reproduced figures below are online-query numbers, distinct from the paper's published full-BAY figures, which are cited as such.

How it works

Precompute the MEPFV once (partition → FPPV → boundary multigraph → BPPV); answer each query by combining FPPV ⊕ BPPV ⊕ FPPV, with MBOR-Adv pruning dominated boundary-node pairs via 2D cost-intervals.

Reproduce it

git clone https://github.com/arunshar/mbor && cd mbor/rust
cargo test                          # exactness on the paper toy + grids
cargo build --release
bash ../data/fetch_dimacs_bay.sh    # vendor open DIMACS Bay Area maps
./target/release/mbor-bench ../data/maps/BAY20-road-d.txt \
    ../data/queries/BAY20-queries 50 3
